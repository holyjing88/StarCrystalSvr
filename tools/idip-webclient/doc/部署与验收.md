# idip-webclient 部署与验收

> **位置**：`starcrystalsvr/tools/idip-webclient/`（服务端仓库，与 `cmd/api` 同仓）  
> **用途**：StarCrystal IDIP 运营台（登录、games.json、H5 上传、审计、金币/任务 IDIP）

---

## 1. 目录结构

| 路径 | 说明 |
|------|------|
| `tools/idip-webclient/` | Vite + TS 源码、`npm run dev` / `build` / `regression` |
| `tools/idip-webclient/doc/` | 产品设计、用例对照 |
| `tools/scripts/idip-webclient/deploy-idip-webclient.sh` | **Linux 一键构建 + nginx + 回归** |
| `tools/scripts/idip-webclient/run-regression.sh` | Vitest 26 条 |
| `tools/scripts/idip-webclient/verify-cloud.sh` | 公网 URL 冒烟 |
| `tools/nginx/starcrystal-idip-https.conf` | HTTPS + SPA + `/idip`/`/api` 反代 |
| `tools/nginx/starcrystal-idip-http.conf` | 仅 HTTP（无证书初装） |
| `/var/www/starcrystal-idip/dist` | 生产静态文件（Linux 默认） |

## 3. Linux 部署（内网 / 云服务器）

**前提**：`starcrystalsvr` 已部署，`release/startsvr.sh` 监听 `8080`；MySQL/Redis 正常。

### 3.1 CentOS 7 / glibc 2.17 说明

- **Vite 6 构建**需 Node 18+，建议在 **Windows/macOS 开发机** `npm run build` 后同步 `dist/` 到 Linux（Samba/rsync）。
- Linux 部署脚本检测到已有 `tools/idip-webclient/dist/index.html` 时**默认跳过 npm build**。
- 强制在 Linux 重编：`FORCE_NPM_BUILD=1`（需 Node 18+ 环境，如 Ubuntu 22 / Rocky 8）。
- **Vitest 回归**：CentOS 7 上建议在开发机执行；Linux 脚本 `run-regression.sh` 适用于 Node 18+ 主机。

### 3.2 内网 Linux（默认 HTTP）

```bash
cd /home/holyjing/starcrystalsvr
sudo VERIFY_HOST=192.168.75.99 bash tools/scripts/idip-webclient/deploy-idip-webclient.sh
# 或显式：ENABLE_HTTPS=0 CLOUD_PUBLIC_HOST=192.168.75.99 sudo bash ...
```

脚本会：

1. 使用已有 `dist/` 或 `npm run build`  
2. 复制 `dist/` → `/var/www/starcrystal-idip/dist`  
3. 安装 `starcrystal-idip-http.conf`（**:80 HTTP**，无 301 跳 HTTPS）  
4. 部署冒烟：`http://<内网IP>/` + `/idip` 反代 + `/healthz`

### 3.3 外网云（必须 HTTPS）

安全组 / 防火墙放行 **80、443**（**不要**对公网开放 **8080**）。

```bash
sudo CLOUD_DEPLOY=1 ENABLE_HTTPS=1 \
  CLOUD_PUBLIC_HOST=203.0.113.10 \
  bash tools/scripts/idip-webclient/deploy-idip-webclient.sh
```

浏览器：`https://<公网IP>/` → 登录 `ops_admin`（账号在 `starcrystal.json` `idip.operators`，密码用加密脚本生成 `passwordEnc`）。**API Base 留空**（同源 `/idip`）。

**生成运营密码（写入配置）**

| 平台 | 命令 |
|------|------|
| Linux | `cd tools/idip-webclient/scripts_encrypt && chmod +x encrypt-idip-operator.sh && ./encrypt-idip-operator.sh` |
| Windows | `.\scripts_encrypt\encrypt-idip-operator.ps1 -ConfigPath …\release\configs\starcrystal.json` |

Linux 脚本内置默认账号 `ops_admin`、密码 `change-me-ops-password`（与 `deploy-env.local.example.sh` 一致）；生产请改脚本内常量或 `export IDIP_OPERATOR_PASSWORD` / `IDIP_OPERATOR_CIPHER_KEY` 后执行。密钥勿提交仓库。

> **说明**：IDIP 中间件校验来源 IP。外网用户须经 **nginx 反代**（API 见 `127.0.0.1`）；勿让运营台直连 `http://公网IP:8080/idip/...`。

### 3.5 卸载（恢复 deploy 前的 nginx / 静态）

```bash
# 绿色部署（停 .deploy 内 nginx，保留文件）
bash tools/idip-webclient/scripts/undeploy-idip-webclient.sh

# 系统 nginx + 恢复被 deploy 改名为 *.bak.deploy-idip.* 的 starcrystal*.conf
sudo bash tools/idip-webclient/scripts/undeploy-idip-webclient.sh --system-nginx

# 并删除 .deploy/idip-webclient 与默认 WEB_ROOT
bash tools/idip-webclient/scripts/undeploy-idip-webclient.sh --purge
sudo bash tools/idip-webclient/scripts/undeploy-idip-webclient.sh --system-nginx --purge
```

不停止 `starcrystalsvr`（8080）；不卸载 nginx 软件包；`/etc/nginx/ssl/starcrystal.*` 自签证书默认保留。

### 3.4 Windows 触发 Linux 部署

```powershell
cd StarCrystal2022
# 内网 Linux：HTTP
.\tools\linux-idip-webclient-deploy.ps1 -LinuxHost 192.168.75.99
# 外网云：HTTPS
.\tools\linux-idip-webclient-deploy.ps1 -LinuxHost 203.0.113.10 -CloudPublicHost 203.0.113.10 -Cloud
```

## 5. nginx / TLS

- 默认自签证书：`/etc/nginx/ssl/starcrystal.crt`（`verify-cloud` / Vitest 需 `-k` 或 `NODE_TLS_REJECT_UNAUTHORIZED=0`）  
- 生产域名：替换为 Let's Encrypt 或云厂商证书，并改 `server_name`  
- 旧「仅 API 反代」模板仍保留：`tools/nginx/starcrystal-https.conf`（无 SPA）
